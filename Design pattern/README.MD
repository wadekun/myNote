# design pattern summary
> 在GoF的《Design Patterns: Elements of Reusable Object-Oriented Software》给出了三类：（创建型【对类的实例化过程的抽象化】），（结构性【描述如何将类或对象结合在一起形成更大的结构】），（行为性【对在不同对象之间划分责任和算法的抽象化】）共23种设计模式，包括：  
1. Abstract Factory(抽象工厂模式)
2. Builder(建造者模式)
3. Factory Method(工厂方法)
4. Prototype(原始原型模式)
5. Singleton(单例模式)
6. Facade(门面模式)
7. Adapter(适配器模式)
8. Bridge(桥梁模式)
9. Composite(组合模式)
10. Decorator(装饰者模式）
11. Flyweight(享元模式)
12. Proxy(代理模式)
13. Command（命令模式）
14. Interpreter(解释器模式)
15. Visitor（访问者模式）
16. Iterator（迭代子模式）
17. Mediator（调停者模式）
18. Memento（备忘录模式）
19. Observer(观察者模式)
20. State（状态模式）
21. Strategy(策略模式)
22. Template Method(模板方法模式)
23. Chain Responsibility(责任链模式)

> 所谓设计模式，就是一套反复被使用代码经验的总结。
